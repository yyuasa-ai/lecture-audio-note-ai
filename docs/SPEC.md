# 仕様書 (SPEC.md)

## 1. アプリケーション名
講義音声ノートAI (Lecture Audio Note AI)

## 2. アプリの目的と概要
このアプリは、大学の講義や自習で録音した「短い音声ファイル」をアップロードすると、AI（Gemini API）を使って自動で**「文字起こし（テキスト化）」「3行での要約」「要点（3〜5個）」**を作成し、ノート整理や復習を助けてくれるツールです。

## 3. 技術の仕組み（システム構成）
初心者でも分かりやすく、研修の提出がしやすいシンプルな仕組みにしています。
* **HTML / CSS / JavaScript のみで作成**: 面倒なサーバーの設定は一切不要です。ブラウザだけで動きます。
* **GitHub Pages で公開可能**: 作成したファイルをそのままインターネット上に無料で公開できます。
* **APIキーを安全に管理する仕組み**: 
  * AIを使うための秘密の鍵（APIキー）は、プログラムのコードの中に直接書き込みません（直書きすると、他人にキーが盗まれて悪用される危険があるためです）。
  * APIキーは画面から入力し、ブラウザの安全な保存庫（localStorage）に保存して使い回します。

## 4. 画面に必要なもの（見た目のパーツ）
画面には以下の部品（要素）を配置します。
1. **設定コーナー（APIキーの設定）**
   * **Gemini APIキー入力欄**: ユーザーが自分のAPIキーを貼り付けるテキスト入力欄。
   * **保存ボタン**: 入力したAPIキーをブラウザに記憶させるボタン。
   * **削除ボタン**: 記憶させたAPIキーを消去するボタン。
2. **操作コーナー（ファイルの選択と実行）**
   * **講義名入力欄**: 何の講義の音声なのか、タイトルを入力するテキスト入力欄。
   * **音声ファイル選択ボタン**: パソコンやスマホから音声ファイルを選ぶためのボタン。
   * **解析ボタン**: 音声ファイルを読み込み、AIへ送信して解析をスタートするボタン。
3. **結果表示コーナー（AIからの回答表示）**
   * **文字起こし表示エリア**: 音声から書き起こされたすべての文章が表示される場所。
   * **要約表示エリア**: 講義の内容を「3行」で短くまとめた文章が表示される場所。
   * **要点表示エリア**: 大事なポイントが「3〜5個の箇条書き」で表示される場所。
   * **コピーボタン**: 表示された結果（文字起こしや要約）をワンクリックでコピーできるボタン。
4. **エラー・状況表示コーナー**
   * **エラー表示エリア**: 動作中に何か問題が発生したときに、赤い文字などで分かりやすくエラー原因を表示する場所。

## 5. エラーが起きたときの親切な対応
以下のような問題が起きたときは、エラー表示エリアに分かりやすい日本語メッセージを表示します。
* **APIキーが登録されていないとき**
  * 表示メッセージ：「Gemini APIキーが設定されていません。画面上の入力欄から設定してください。」
* **音声ファイルが選ばれていないとき**
  * 表示メッセージ：「解析する音声ファイルを選択してください。」
* **AI（Gemini API）との通信に失敗したとき**
  * 表示メッセージ：「AIとの通信に失敗しました。APIキーが正しいか、またはインターネットの接続を確認してください。」
* **AIから返ってきた結果の整理に失敗したとき（JSONエラーなど）**
  * 表示メッセージ：「AIの回答を正しく処理できませんでした。もう一度お試しいただくか、プロンプトの設定を確認してください。」

## 6. 使うときの注意点
* **個人情報や秘密の情報は入れない**: AIサービスに音声を送信するため、名前や住所などの個人情報、または公開されては困る秘密の情報が含まれる音声は使用しないでください。
* **最初は短い音声で試す**: AIの処理には時間がかかることがあります。まずは **30秒〜2分程度** の短い音声ファイルを使って、正しく動くかテストしてください。
